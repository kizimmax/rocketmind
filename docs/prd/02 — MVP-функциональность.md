# 02 — MVP-функциональность

> Источник: PRD Rocketmind AI Agent MVP 1.1 (февраль 2026)

---

## Назначение

Документ фиксирует требования к упрощённой версии MVP-1.1 web-сервиса ИИ-агентов.
Основной фокус: **«кейсы (проекты) → чаты с ИИ-агентами»** с интеграцией логики через n8n.

---

## Рамки итерации (Scope)

| Параметр | Значение |
|---|---|
| Итерация | MVP-1.1 (февраль–март 2026) |
| Канал | Web-only |
| Онбординг | Исключён. Вход напрямую в интерфейс чата после авторизации |
| Живые эксперты | Исключены. Взаимодействие только с ИИ-агентами |
| Платёжный UI | Без отдельных экранов. Ссылки/триггеры оплаты — через сообщения и кнопки в чате (логика n8n) |

---

## Цель итерации

Предоставить пользователю инструмент для ведения нескольких «кейсов» (проектов), внутри которых он ведёт диалоги с различными ИИ-агентами.

Стабильный цикл: **авторизация → выбор агента → диалог → получение результата/ссылки на оплату**.

---

## Success Criteria (Приёмка)

Критический путь работает стабильно:

1. **Авторизация** — Login 2-step (email + code)
2. **Автобутстрап** — По URL `/a/{agent_slug}` автоматически создаётся кейс и открывается чат
3. **Диалог** — Отправка/получение сообщений от ИИ-агента через вебхуки n8n
4. **Результат** — Итоговое summary или ссылка на оплату/форму заявки как системное сообщение

---

## Экраны и UI-структура

| # | Экран | Описание |
|---|---|---|
| 1 | Лендинг сервиса | CTA-кнопка «Попробовать» |
| 2–3 | Авторизация | Поле Email → ввод кода из письма |
| 4 | Главный интерфейс | Сайдбар (кейсы + агенты) + центральная область (чат) |
| 5 | Каталог агентов | Выбор новых ИИ-агентов для добавления в кейс |

### Главный интерфейс (детали)
- **Левый сайдбар:** управление кейсами (создание, переименование, архив) + список активных агентов в кейсе
- **Центральная область:** окно чата с ИИ-агентом

---

## Функциональные требования

### Сценарий взаимодействия
1. Пользователь авторизуется и выбирает кейс (или заходит по прямой ссылке на агента)
2. Ведёт диалог в чат-интерфейсе
3. Получает результат в виде текстовых сообщений или резюме (summary)

### Интеграция n8n
- Все сообщения пользователя уходят на вебхук n8n
- **Триггеры оплаты:** при достижении условий сценария n8n отправляет системное сообщение со ссылкой на оплату

### Кейсы и изоляция
- История диалогов и набор агентов изолированы внутри каждого кейса
- Возможность архивации и восстановления кейсов

---

## Вне рамок спринта (Not Goals)

- Интерфейс «Личного кабинета эксперта»
- Сложные анимации
- Кастомный UI воронки (используется shadcn/ui)
- Прямая интеграция платёжного шлюза
